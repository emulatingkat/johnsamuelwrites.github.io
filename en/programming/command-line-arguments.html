A Relook at Command-Line Arguments

John Samuel

NOTE: Article in Progress

Command-line interface is one of the oldest, still commonly used human-computer interface, where users type textual commands with arguments to get desired results. It is still common to see developers providing command-line options for software applications along with graphical user interface and in order to cater to this common requirement, several programming languages provide standard libraries to help the programmers parse the command-line arguments. However, both CLI and libraries have evolved all these years, standardizing the way how commands are built adding along with new functionalities like the use of colors, auto-completion, auto-correction etc. This article explores some of these advances and discusses how command line interface can be further enhanced, especially considering some of the recent research developments.

Introduction

Command line can be considered as one of the oldest human-computer interface, where you type in some textual commands and the computer responds with a result. Most of these commands are abbreviations of their longer forms related to a key functionality (usually in English). Take for example, Unix/Linux command $ ls ls is an abbreviation of list and its main function is to list files in a given directory. Command-line or terminal commands, hereafter referred to as simply commands.
